Definition Tables

StatsDef
Stats definition table.

Fields:

  • hp int Full health level (1 HP = 1/2 player heart).
  • hostile (optional) bool Is mob hostile (required for mode "attack") (default: false).
  • lifetime (optional) int After which time mob despawns, in seconds.
  • dies_when_tamed (optional) bool Stop despawn when tamed (default: false).
  • can_jump (optional) int Height in nodes (default: 0).
  • can_swim (optional) bool Can mob swim or will it drown (default: false).
  • can_fly (optional) bool Allows to fly (requires mode "fly") and disable step sounds (default: false).
  • can_burn (optional) bool Takes damage of lava (default: false).
  • can_panic (optional) bool Runs fast around when hit (requires mode "walk") (default: false).
  • has_falldamage (optional) bool Deals damage if falling more than 3 blocks (default: false).
  • has_kockback (optional) bool Get knocked back when hit (default: false).
  • sneaky (optional) bool Disables step sounds if true (default: false).
  • light (optional) table

    Which light level will burn creature (requires can_burn = true).

    Example:

      light = {min=10, max=15}
ModeDef

Modes definition table.

Entity behavior definition. Behavior types are idle, walk, attack, follow, eat, death, & panic. The sum of all modes must be 1.0.

Example:

  modes = {
    idle = {chance=0.3,},
    walk = {chance=0.7, moving_speed=1,},
  }

Fields:

  • chance float Number between 0.0 and 1.0 (*NOTE: sum of all modes MUST be 1.0*). If chance is 0 then mode is not chosen automatically.
  • duration int Time in seconds until the next mode is chosen (depending on chance).
  • moving_speed (optional) int Moving speed (walking/flying/swimming).
  • update_yaw (optional) int Timer in seconds until the looking dir is changed. If moving_speed > 0 then the moving direction is also changed.
  • radius int (follow & eat modes only) Search distance in blocks/nodes for player.
  • timer int (follow & eat modes only) Time in seconds between each check for player.
  • items table (follow & eat modes only) Table of items to make mob follow in format {<Itemname>, <Itemname>}; e.g. {"farming:wheat"}.
  • nodes table (eat mode only) Eatable nodes in format {<Itemname>, <Itemname>}; e.g. {"default:dirt_with_grass"}.
ModelDef
Model definition table.

Fields:

  • mesh string Mesh name (see Minetest Documentation for supported filetypes).
  • textures table Table of textures (see Minetest Documentation).
  • collisionbox NodeBox Defines mesh collision box (see Minetest Documentation).

  • backface_culling (optional) bool Set to true to enable backface culling.
  • animations (optional) table Table of AnimationDef used if defined.
AnimationDef

Animations defiintion table.

Animations coincide with modes. E.g. idle, walk, etc.

Example:

  animations = {
    idle = {start=25, stop=75, speed=15,},
    walk = {start=75, stop=100, speed=15,},
  }

Fields:

  • start int Start frame.
  • stop int End frame.
  • speed int Animation speed.
  • loop (optional) bool If false, animation if just played once (default: true).
  • duration (optional) int (death mode only) Sets time the animation needs until mob is removed.
SoundsDef

Sounds definition table.

Sounds can be defined for these actions: *on_damage*, *on_death*, swim, & random.

random is a table of SoundDef that will be played randomly during the modes for which they are set.

Example:

  sounds = {
    on_damage = {name="creatures_horse_neigh_02", gain=1.0},
    on_death = {name="creatures_horse_snort_02", gain=1.0},
    random = {
      idle = {name="creatures_horse_snort_01", gain=1.0},
      follow = {name="creatures_horse_neigh_01", gain=1.0, time_min=10},
    },
  }

Fields:

  • on_damage (optional) SoundDef Sound played when entity is hit.
  • on_death (optional) SoundDef Sound played when entity dies.
  • swim (optional) SoundDef Sound played while entity is swimming.
  • Random (optional) table Sounds that will play randomly during specified modes. E.g. idle, walk, etc.
SoundDef
Sound definition.

Fields:

  • name string Sound file name without file type extension (e.g. "mysound", not "mysound.ogg") (see Minetest documentation).
  • gain float Sound gain (see Minetest documentation).
  • distance (optional) int Distance in blocks/nodes at which sound can be heard.
  • time_min (optional) int (random mode only) Minimum time in seconds between sounds.
  • time_max (optional) int (random mode only) Maximum time in seconds between sounds.
DropDef

Item drops definition table.

Example:

  drops = {
    {"default:wood"}, -- 1 item with 100% chance
    {"default:wool", 1, chance=0.3}, -- 1 item with 30% chance
    {"default:stick", {min=2, max=3}, chance=0.2}, -- between 2-3 items with 20% chance
  }
CombatDef
Combat definition table.

Fields:

  • attack_damage int How much damage is dealt on each hit.
  • attack_speed (optional) float Time in seconds between hits (default: 1.0).
  • attack_radius float Distance in blocks mob can reach to hit.
  • search_enemy bool true to search enemies to attack.
  • search_timer int Time in seconds to search an enemy (only if none found yet).
  • search_radius int Radius in blocks within enemies are searched.
  • search_type string What enemy is being searched (see types at cmer.findTarget).
SpawnDef
Spawning definition table.

Fields:

  • abm_nodes ABMNodesDef On what nodes mob can spawn.
  • abm_interval int Time in seconds until Minetest tries to find a node with set specs.
  • abm_chance int Chance is 1/.
  • max_number int Maximum mobs of this kind per mapblock (16x16x16).
  • number int How many mobs are spawned if found suitable spawn position. Can be int or table: number = {min=<value>, max=<value>}
  • time_range (optional) table Time range in time of day format (0-24000) (table with min & max values).
  • light (optional) table Min and max lightvalue at spawn position (table with min & max values).
  • height_limit (optional) table Min and max height (world Y coordinate) (table with min & max values).
  • spawn_egg (optional) table Is set a spawn_egg is added to creative inventory (table with description & texture values).
  • spawner (optional) SpawnerDef Is set a spawner_node is added to creative inventory.
ABMNodesDef
ABM nodes definition table.

Fields:

  • spawn_on (optional) table List of nodes on which the mob can spawn.
  • neighbors table List of nodes that should be neighbors where mob can spawn. Can be nil or table as above. "air" is forced always as neighbor.
SpawnerDef
Spawner definition table.

Fields:

  • range int Defines an area (in blocks/nodes) within mobs are spawned.
  • number int Maxmimum number of mobs spawned in area defined via range.
  • description (optional) string Item description as string.
  • light (optional) table Min and max lightvalue at spawn position.
